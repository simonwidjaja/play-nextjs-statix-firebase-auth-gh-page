Using the Typo Animation Feature

Typo animations are pretty useful to present text content in a more compelling way. Yet creating really compelling typo animations are usually pretty time consuming, which is why you often revert to basic “fade-in” or move in animations for the entire text element. Since you see that all over the place, the user is hardly captivated by that.

Now with the Typo animation feature in the Toolbelt you can apply animations to each letter of a complete text field simply by naming that (dynamic) text field in a certain way. Here’s an example:

See how the text appears letter by letter with a random scramble animation before the actual letter is presented. Let’s see how this is done…

Understanding the project

Open the AN file in the 010 Start folder. We will start with an Animate project that already contains the text fields and the animation for the picture in the background. The headline text has a simple scale animation. The second text has no animation at the moment.

Create scramble animation for subheadline text

We will start with the second text field as it is not encapsulated in a symbol and therefore a bit easier. Simply select the text field on the Stage (the one that says “London - Piccadilly Circus”).

If you take a look at the property inspector you can see that it is a dynamic text field. The Typo Animation effect does ONLY work with dynamic text fields (because they preserve the text as text and offer an instance name to identify them).

We will use that very instance name now to tell the Toolbelt what it should do for us. In this case it is TA_TYPE_SCRAMBLE_DELAY_2000_SPEED_450

Now let’s take a look at the different parts here:

  • Every letter needs to be in caps and use an underscore to separate words
  • Start with TA to define the animation kind: Typo Animation
  • Optionally you can then define the type, delay and speed, using the keywords followed by an underscore and the value (type = name, delay and speed = number in milliseconds)

If you don’t set type, duration or speed it will use default values (for example “appear” as type).

Note that the animation is not visible in Animate itself (during design time). You need to preview the project in a browser to see the animation take place. We will do that after the next step.

Create scramble animation for headline text

Now to show you that the Toolbelt Typo Animation does also work in nested text elements and therefore can be combined with “classic” animations in Animate we will add the effect now to the headline as well. The headline text is placed in a graphic symbol (in order to create the scale animation). Double click on the headline text to access the symbol scope (in this case “Tween 1”).

Now you can again use the Instance Name in the Properties Inspector to apply a Typo Animation to this element. It should start earlier with less speed. So set the name to

TA_TYPE_SCRAMBLE_DELAY_900_SPEED_250

Of course you can try different values and types to get a feeling for it.

Preview the result in the browser

Now preview the project in the browser to see the animations in action (Cmd/Ctrl+Enter or Control > Test in the menu). If you followed the instructions carefully, the text will now appear letter by letter with a scramble animation.

Happy Animating!

Good to know… Remember to make sure that your export settings don’t export all elements as texture (use sprite sheet instead). Otherwise the elements can not be modified at runtime. Also note that Animate loops the timeline per default. So if you set the Toolbelt animation at the end of the timeline without using a stopper for the timeline, you might not see any animations (as Animate restarts the timeline before the Toolbelt animation takes place).
